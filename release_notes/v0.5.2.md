# PyWiki v0.5.2 — Release Notes

**Released:** 2026-03-05
**Branch:** `devel`
**Tag:** `v0.5.2`

---

## Highlights

This release focuses on MediaWiki migration quality-of-life improvements and fixes
several intermittent bugs encountered when standing up new server instances.

---

## What's New

### Back button after page create

When creating a new page by following a red link (or clicking Create Page from a wiki
page), a one-shot **← Back** button now appears in the page header after saving.
It links back to the page you navigated from and disappears after one view.

The source URL is captured from the HTTP `Referer` header, but only for plain wiki
page views — edit, history, move, diff, and create pages are excluded to avoid
capturing a stale URL.

### Upload form pre-fill from missing image links

`[[File:name.png]]` and `[[Image:name.png]]` links to non-existent attachments now
render as clickable links directly to `/special/upload`, pre-filled with the filename,
namespace, page slug, and a `back` URL. After a successful upload a **← Back to page**
button is shown so you return to the page that had the missing image.

### `[[Image:...]]` alias

`[[Image:filename.ext]]` is now fully supported as an alias for `[[File:filename.ext]]`
in the wikitext renderer, matching MediaWiki behaviour.

---

## Bug Fixes

### Page-not-found race condition (create and edit)

`create_page_submit` and `edit_page_submit` were missing `await db.commit()` before
the `303` redirect. The browser followed the redirect before the session committed,
causing intermittent *"The page X does not exist yet"* errors immediately after saving.
Both routes now commit explicitly before redirecting (matching the namespace routes
fixed in v0.5.1).

### `Category` namespace becoming the default

The `pref_namespace` cookie was set unconditionally after every page save, including
saves in the `Category` namespace. On new installs where an admin creates the Category
namespace first, it would become the default. The cookie is now never set to `Category`.
The ⭐ Set default button on `/special/namespaces` has the same guard.

### Title capitalisation destroys acronyms

The "Create this page" link on the 404 page applied Jinja2's `| title` filter to the
slug, converting `perl` → `Perl` and `mq` → `Mq`. The filter has been removed; the
slug is now only de-hyphenated, preserving whatever capitalisation the user types in
the create form.

### Bare URL auto-linking at end of line

The wikitext bare-URL regex required a trailing space, so URLs at the end of a line
were not linked. The pattern now uses a lookahead `(?=[\s<>'"]|$)` to correctly match
end-of-line URLs.

### Startup seeding failures were silent

`_seed_defaults()` swallowed all exceptions. Failures now log via `log.exception()` so
they appear in `journalctl`. Successful seeding logs at INFO level so fresh installs
are easy to verify.

## Upgrade Instructions

```bash
cd /opt/pywiki
git pull origin devel
systemctl stop pywiki && systemctl start pywiki
```

No database migrations required for this release.

Verify startup seeding ran correctly:
```bash
journalctl -u pywiki -n 20 --no-pager | grep -iE 'seed|error|exception'
```

---

## Known Limitations

- The `← Back` button after page create relies on the HTTP `Referer` header; browsers
  that strip the Referer (strict privacy mode) will not show the button.
- Missing-file upload links are enriched in `view_page` only — the live preview does
  not show them as clickable links.
