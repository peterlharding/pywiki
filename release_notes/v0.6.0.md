# PyWiki v0.6.0 — Release Notes

**Released:** 2026-03-06
**Branch:** `devel`
**Tag:** `v0.6.0`

---

## Highlights

This release adds full **math / LaTeX rendering** support via KaTeX across all three
content formats (Markdown, RST, Wikitext), using the KaTeX v0.16.11 CDN with
client-side auto-rendering.

---

## What's New

### Math rendering (KaTeX)

LaTeX expressions are now rendered as formatted mathematics in the browser.

#### Wikitext

Use the standard MediaWiki `<math>` tag:

```
Inline: <math>E = mc^2</math>

Display block:
<math display="block">\frac{-b \pm \sqrt{b^2 - 4ac}}{2a}</math>
```

The server converts `<math>` tags to KaTeX `\(...\)` / `\[...\]` delimiters before
the page is sent to the browser.

#### Markdown

Use standard `$...$` (inline) and `$$...$$` (display) delimiters:

```markdown
The formula $x^2 + y^2 = z^2$ is well known.

$$
\int_0^\infty e^{-x^2}\,dx = \frac{\sqrt{\pi}}{2}
$$
```

These pass through the Markdown renderer unchanged and are processed client-side
by KaTeX auto-render.

#### RST

Use the standard docutils `:math:` role and `.. math::` directive:

```rst
The formula :math:`E = mc^2` is famous.

.. math::

   \frac{d}{dx}\left(\int_a^x f(t)\,dt\right) = f(x)
```

RST math is pre-processed **before** docutils so the LaTeX source is preserved and
rendered by KaTeX rather than converted to MathML.

## Technical details

- KaTeX `v0.16.11` loaded from `cdn.jsdelivr.net` — CSS in `<head>`, JS + auto-render
  deferred at end of `<body>`. Requires internet access for first-time page loads;
  browsers cache the CDN assets.
- Auto-render ignores `<pre>`, `<code>`, `<script>`, `<style>` content, and any
  element with class `no-math`.
- `RENDERER_VERSION` bumped to `12` — all stale cached page HTML is automatically
  re-rendered on next view (self-healing cache).
- 9 new tests in `tests/test_17_math.py` — **278 tests total**.

## Upgrade Instructions

```bash
cd /opt/pywiki
git pull origin devel
systemctl stop pywiki && systemctl start pywiki
```

No database migrations required for this release.

The RENDERER_VERSION bump means all cached page renders will be automatically
invalidated and re-rendered on first view after the restart.
