# PyWiki v0.6.1 — Release Notes

**Released:** 2026-03-06
**Branch:** `devel`
**Tag:** `v0.6.1`

---

## Highlights

Bug-fix release addressing RST category handling, the footer version display,
the missing `/api/v1/render` POST handler, and a `PORT` → `APP_PORT` rename
in the Makefile and `.env.example`.

---

## Fixes

### RST category tags not recognised

`.. category:: Name` directives in RST pages were silently ignored — pages did
not appear in the category index and no category footer was shown.

Three locations were affected:

- **`extract_categories()`** (`renderer.py`) — RST format now checks both
  `.. category:: Name` and the wikitext-style `[[Category:Name]]` fallback.
- **`get_pages_in_category()`** (`pages.py`) — RST pages are now matched using
  the `.. category::` regex so they appear under `/category/<Name>`.
- **`special_status` category scan** (`views.py`) — the `ilike` pre-filter now
  includes `%.. category::%` so RST pages are counted in `/special/categories`.

Both syntaxes work in RST pages:

```rst
.. category:: Programming

[[Category:Python]]
```

### Footer showing wrong version (`v1.0.0`)

`APP_VERSION=1.0.0` hardcoded in `.env` was overriding the automatic version
read from `pyproject.toml`. Fixes:

- `APP_VERSION` removed from `.env.example` — new installs won't be affected.
- `app/_version.py` now always reads `pyproject.toml` first; falls back to
  `importlib.metadata` only when the source tree is not present (e.g. installed
  as a wheel without source).

**Action required on existing servers:** remove `APP_VERSION=...` from
`/opt/pywiki/.env` then restart the service.

```bash
sed -i '/^APP_VERSION=/d' /opt/pywiki/.env
systemctl stop pywiki && systemctl start pywiki
```

### `/api/v1/render` missing POST handler

The render endpoint docstring declared `POST` but only a `GET` handler was
registered. Large page previews could be silently truncated by URL length
limits in proxies or browsers. A proper `POST` handler accepting a JSON body
is now the primary path:

```
POST /api/v1/render
Content-Type: application/json

{"content": "...", "format": "markdown", "namespace": "Main", "slug": ""}
```

The `GET` endpoint is retained for backward compatibility.

### `APP_PORT` rename

`PORT` renamed to `APP_PORT` in `Makefile` and `.env.example` to avoid
collision with system environment variables and make the purpose clearer.

**Action required:** update your local `.env` — change `PORT=8000` to
`APP_PORT=8000`.
