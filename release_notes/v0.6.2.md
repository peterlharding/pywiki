# PyWiki v0.6.2 — Release Notes

**Released:** 2026-03-06
**Branch:** `devel`
**Tag:** `v0.6.2`

---

## Highlights

Observability and developer-experience release: health check UI page,
in-memory log review, improved `/api/health` endpoint, and cleaner
pytest output.

---

## Added

### `/special/health` — Health check UI page

A human-readable health check page available to all users at
`/special/health`.  Shows:

- Overall status (OK / Degraded)
- App name, version, renderer version, environment
- Database connectivity status and round-trip latency
- Link to the raw JSON endpoint `/api/health`

### `/special/logs` — Log review page (admin-only)

An in-memory ring buffer capturing the last 500 log records (INFO and
above) from all loggers.  Accessible at `/special/logs` by admins only.

- Level filter dropdown (defaults to WARNING to hide access log noise)
- Rows colour-coded by severity (ERROR/CRITICAL = red, WARNING = amber)
- Resets on process restart — noted clearly on the page
- Linked from Special Pages under the admin section

### `/api/health` improvements

The JSON health endpoint now:
- Probes the database with `SELECT 1` and reports `latency_ms`
- Includes `renderer_version` in the response
- Returns HTTP `503` with `"status": "degraded"` if the DB is unreachable

```json
{
  "status": "ok",
  "app": "PyWiki",
  "version": "0.6.2",
  "renderer_version": 12,
  "database": { "status": "ok", "latency_ms": 1.4 }
}
```

### `make test-v` target

New Makefile target for verbose per-test output.  `make test` now shows
compact dot-per-test progress by default (removed `-v` and `log_cli`
from `pyproject.toml`).

---

## Fixed

### Log buffer empty on first visit

`logging_buffer.py` previously installed itself only during the FastAPI
lifespan startup hook.  Under uvicorn `--reload` the timing was
unreliable.  The handler now auto-installs at module import time,
ensuring records are captured from the moment the process starts.

---

## Upgrade Instructions

```bash
cd /opt/pywiki
git pull origin main
systemctl stop pywiki && systemctl start pywiki
```

No database migrations required for this release.
