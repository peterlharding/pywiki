# PyWiki v0.6.5 — Release Notes

**Released:** 2026-03-07
**Branch:** `devel`
**Tag:** `v0.6.5`

---

## Highlights

Dark mode, breadcrumb navigation, and several bug fixes.

---

## Added

### Dark mode

PyWiki now supports dark mode automatically and with a manual toggle.

- **Automatic** — respects `prefers-color-scheme: dark` OS/browser setting.
- **Manual** — 🌙 / ☀️ toggle button in the navbar; preference persisted in `localStorage`.
- **Flash-free** — an inline `<script>` on `<html>` applies the saved theme before CSS loads.
- Theme tokens use CSS custom properties throughout; no hardcoded colours remain.
- Three theme states: `[data-theme="dark"]`, `[data-theme="light"]`, and auto (follows OS).

Colour palette (dark): Catppuccin Mocha-inspired — dark blue-grey backgrounds, soft
text, blue accent links, muted greens/reds for success/danger.

### Breadcrumb navigation

Every page view, history, diff, move, and namespace index page now shows a
breadcrumb trail:

```
Home › Main › My Page › History
```

- Implemented via a `{% block breadcrumb %}` slot in `base.html`.
- CSS `.breadcrumb` class in `wiki.css`.

## Fixed

### Namespace export 500 error

`GET /wiki/{namespace}/export` returned a 500 Internal Server Error because
`sqlalchemy.func` was not imported in `views.py`. Fixed by adding the import.

### Namespace edit form pre-fill

The description and default-format fields on the namespace edit form did not
always show the saved values on re-visit. Root cause: Jinja2 `or` operator
treats empty string as falsy. Fixed using `is not none` sentinel check.

### Dead `/admin` link on Special Pages

The Special Pages admin section linked to `/admin` which returned 404 (that
path is API-only under `/api/v1/admin/`). Replaced with `/special/namespaces`.

```bash
git pull origin main
systemctl stop pywiki && systemctl start pywiki
```
