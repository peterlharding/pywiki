# PyWiki v0.6.6 — Release Notes

**Released:** 2026-03-07
**Branch:** `devel`
**Tag:** `v0.6.6`

## Highlights

Colour utility classes for wiki content, and a dark mode cache-busting fix for servers.

---

## Added

### Colour utility classes

13 CSS classes for colouring text in any markup format, all dark-mode aware:

| Class | Colour |
|---|---|
| `.text-red` | Red (adapts in dark mode) |
| `.text-green` | Green |
| `.text-blue` | Blue |
| `.text-orange` | Orange |
| `.text-purple` | Purple |
| `.text-teal` | Teal |
| `.text-grey` | Grey |
| `.text-gold` | Gold |
| `.text-muted` | Theme muted (`--muted`) |
| `.text-accent` | Theme accent (`--accent`) |
| `.text-danger` | Theme danger (`--danger`) |
| `.text-success` | Theme success (`--success`) |
| `.text-warn` | Theme warning (`--warn`) |

Usage (works in Markdown, Wikitext, and RST via `.. raw:: html`):

```html
<span class="text-red">Red text</span>
<p class="text-teal">Teal paragraph</p>
<h2 class="text-accent">Accent heading</h2>
```

Full usage guide: `docs/colour-text.md`

## Fixed

### Dark mode toggle on production servers

The `wiki.js` script tag had no cache-busting version string, so browsers and
nginx served the old (pre-dark-mode) JS file indefinitely. Added
`?v={{ app_version }}` to the `<script>` tag — matching the existing pattern
used by `wiki.css` and `pygments.css`.

---

## Fixed

### Redirect deletion

After moving a page, the redirect stub left at the old URL can now be deleted
from the UI:

1. Navigate to the old redirect URL with `?redirect=no` appended, e.g.
   `/wiki/Main/old-title?redirect=no`
2. Click the **🗑️ Delete redirect** button in the page actions bar
3. Confirm — the stub is permanently deleted and you are returned to the namespace index

The button is only shown to logged-in users and only on redirect pages.

### Dark mode toggle on servers

`wiki.js` had no cache-busting version query string, so browsers and nginx
served the old JS indefinitely. Added `?v={{ app_version }}` to match the
existing pattern on `wiki.css`.
