# Release Notes — v0.6.7

**Date:** 2026-03-08

## Overview

v0.6.7 completes the export/import workflow, making it practical to transfer a
selected subset of pages (with all their attachments) between wiki instances —
for example from a development instance used for content iteration to a
production server.

---

## New Features

### Selective Export

Previously the only export option was "Export all" which downloaded every page
in a namespace as a ZIP. You can now select individual pages:

- **Namespace index** — each row in the page table has a checkbox. Tick the
  pages you want, then click **↓ Export selected**. A "Select all" toggle with
  indeterminate state is provided. The resulting ZIP contains only the chosen
  pages plus their attachments.
- **Search results** — logged-in users see a checkbox next to each result.
  Selections can span multiple namespaces; clicking **↓ Export selected**
  posts to `POST /special/export/selected` which groups pages by namespace and
  builds a single multi-directory ZIP.

Routes added:
- `POST /wiki/{namespace}/export/selected`
- `POST /special/export/selected`

### ZIP Import

A new import form in the **namespace index sidebar** accepts a `.zip` file and
upserts its contents into the namespace:

**Pages** (`.md`, `.rst`, `.wiki` files):
- If the slug does not exist → page is **created**.
- If the slug already exists → a **new version** is added (full history
  preserved).
- The page title is derived from the slug (hyphens replaced with spaces,
  title-cased) on creation; on update the existing title is kept.

**Attachments** (`{dir}/{slug}/attachments/{filename}` entries):
- File is written to the attachment storage directory using the same path
  convention as a normal upload (`{namespace}/{slug}/{filename}`).
- An `Attachment` DB record is **created** if the filename is new for that
  page, or **updated** (size, content type, file on disk) if it already
  exists.

After import a result banner appears:

> Import complete — **N** page(s) created, **M** updated; **P** attachment(s)
> imported, **Q** updated.

Route added: `POST /wiki/{namespace}/import`

## Changed

- **"Export ZIP" button** renamed to **"Export all"** on the namespace index to
  distinguish it from the new selective export button.
- `export_namespace` refactored to share a `_build_zip()` helper with the
  selective export routes.

## Upgrade Notes

No database schema changes. No new dependencies. No configuration changes
required.

Standard deploy procedure:

```bash
cd /opt/pywiki
git pull
systemctl stop pywiki
systemctl start pywiki
```
