# Release Notes — v0.6.8

**Date:** 2026-03-08

## Overview

v0.6.8 adds the ability to delete a page directly from the editor UI, removing
the need to use the API manually.

---

## New Features

### Delete page from editor

A red 🗑 **Delete page** button now appears at the bottom-right of the edit
form (`/wiki/{namespace}/{slug}/edit`), visually separated from Save / Cancel.

- Clicking the button opens a browser confirmation dialog:
  *"Permanently delete 'Page Title' and all its history? This cannot be
  undone."*
- On confirmation, a `POST /wiki/{namespace}/{slug}/delete` request is
  submitted.
- The page, all its versions, and all attachment records are deleted (cascaded
  by the DB foreign keys).
- The user is redirected to the namespace index (`/wiki/{namespace}`).
- Login is required — unauthenticated attempts are redirected to `/login`.
- Attempting to delete a non-existent slug returns 404.

---

## Fixed

- **Delete button had no effect on first deploy** — the delete `<form>` was
  accidentally nested inside the edit `<form>`. Browsers silently discard
  nested forms and submit only the outermost one, so clicking Delete was
  saving the page instead. The delete form has been moved outside the edit
  form.
