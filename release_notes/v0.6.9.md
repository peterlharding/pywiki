# Release Notes — v0.6.9

**Date:** 2026-03-09

## Overview

v0.6.9 adds a password show/hide toggle to every password field in the UI,
improving usability when entering or verifying credentials.

---

## New Features

### Password show/hide toggle

All password inputs now have a 👁 eye button overlaid on the right edge of
the field:

- Click 👁 to reveal the password as plain text — the icon swaps to 🙈.
- Click 🙈 to hide it again.

Applies to all five password fields:

| Page | Field |
|------|-------|
| Login | Password |
| Register | Password |
| Reset password | New password, Confirm new password |
| Admin → Create user | Password |
| Admin → Edit user | New password |

**Implementation:**
- `.pw-wrap` / `.pw-toggle` CSS classes in `wiki.css` — the wrapper is
  `position:relative; display:block; width:100%`; the button is
  `position:absolute` inset to the right edge with `z-index:1` so it overlays
  the input border.
- A single event-delegated `click` handler in `base.html` — no per-template
  JavaScript. Toggling flips `input.type` between `"password"` and `"text"`.

---

## Upgrade Notes

No database schema changes. No new dependencies. No configuration changes
required.

Standard deploy procedure:

```bash
cd /opt/pywiki
git pull
systemctl stop pywiki
systemctl start pywiki
```
